## 📌 What is Windows Defender Firewall?

Windows Defender Firewall (WDF) is a **host-based, stateful firewall** built into Windows.
It filters **inbound and outbound** network traffic based on rules.

**Key Architecture:**
```
Application Traffic
       ↓
┌──────────────────�┐
│   WFP Engine      │  ← Windows Filtering Platform (kernel)
│   (netio.sys)     │
├──────────────────┤
│   BFE Service     │  ← Base Filtering Engine (manages filters)
├──────────────────┤
│   MPSSVC Service  │  ← Microsoft Protection Service (firewall service)
├──────────────────┤
│   Firewall Rules  │  ← What YOU configure
└──────────────────┘
```

**Critical Services:**
- **BFE** (Base Filtering Engine): Must be running. If stopped → all filtering stops
- **MPSSVC** (Windows Defender Firewall service): Manages firewall rules/profiles

## 📌 Three Firewall Profiles

| Profile | When Active | Typical Setting |
|---------|------------|-----------------|
| **Domain** | Machine connected to AD domain network | Less restrictive |
| **Private** | User marks network as "private/home" | Moderate |
| **Public** | Default for unknown networks | Most restrictive |

**⚠️ Key Point**: A machine can have **multiple profiles active simultaneously** (e.g., Domain on Ethernet + Public on Wi-Fi). Each NIC maps to one profile.

## 📌 Inbound vs Outbound Rules

- **Inbound Rules**: Control traffic **coming INTO** this machine
  - Default behavior: **BLOCK** (unless a rule allows it)
- **Outbound Rules**: Control traffic **going OUT FROM** this machine
  - Default behavior: **ALLOW** (unless a rule blocks it)

**Rule Components:**
| Component | What it does | Example |
|-----------|-------------|---------|
| Program | Match by executable path | `C:\Program Files\App\app.exe` |
| Port | Match by port number | TCP 443, UDP 53 |
| Protocol | Match by protocol | TCP, UDP, ICMPv4 |
| Scope | Match by IP range | `10.0.0.0/8` |
| Action | Allow or Block | Allow / Block |
| Profile | Which profile applies | Domain, Private, Public |

## 📌 Essential Commands (Your Daily Tools)

### Check Firewall Status
```powershell
# PowerShell
Get-NetFirewallProfile | Format-Table Name, Enabled, DefaultInboundAction, DefaultOutboundAction

# Netsh
netsh advfirewall show allprofiles
```

### List Firewall Rules
```powershell
# All enabled rules
Get-NetFirewallRule -Enabled True | Format-Table DisplayName, Direction, Action, Profile

# Find rules for a specific program
Get-NetFirewallRule -Enabled True | Where-Object { 
    ($_ | Get-NetFirewallApplicationFilter).Program -like "*sqlserver*" 
}

# Find rules for a specific port
Get-NetFirewallRule -Enabled True | Get-NetFirewallPortFilter | 
    Where-Object { $_.LocalPort -eq "1433" }
```

### Quick Enable/Disable (USE WITH CAUTION)
```powershell
# Disable firewall on all profiles (troubleshooting only!)
Set-NetFirewallProfile -All -Enabled False

# Re-enable
Set-NetFirewallProfile -All -Enabled True
```

## 📌 Where to Find Logs

1. **Event Logs**: `Applications and Services Logs > Microsoft > Windows > Windows Firewall With Advanced Security > Firewall`
2. **Firewall Log File**: Default at `%systemroot%\system32\LogFiles\Firewall\pfirewall.log`
   - ⚠️ Logging is **OFF by default** — must enable it manually
3. **Enable logging**:
```powershell
Set-NetFirewallProfile -All -LogBlocked True -LogAllowed True -LogFileName "%systemroot%\system32\LogFiles\Firewall\pfirewall.log"
```
