# L200: Windows Firewall Troubleshooting
**⏱ Estimated: 10 minutes | 🎯 Goal: Diagnose and resolve common firewall issues**

---

## 📌 Troubleshooting Framework

When customer says "something is blocked / can't connect":

```
Step 1: Is the firewall actually the cause?
         ↓ Yes
Step 2: Which profile is active on the affected NIC?
         ↓
Step 3: Is there a Block rule or missing Allow rule?
         ↓
Step 4: Is the rule from Local Policy or GPO?
         ↓
Step 5: Fix — Add/modify rule or fix GPO
```

---

## 📌 Step 1: Confirm Firewall is the Cause

**Quick Test**: Temporarily disable firewall → does the issue resolve?
```powershell
# Save current state first!
Get-NetFirewallProfile | Select Name, Enabled | Export-Csv C:\temp\fw-state.csv

# Disable (temporary troubleshooting only)
Set-NetFirewallProfile -All -Enabled False

# Test connectivity...

# Re-enable immediately!
Set-NetFirewallProfile -All -Enabled True
```

**⚠️ Warning**: Never leave firewall disabled. If disabling fixes the issue → firewall IS the cause, now find which rule.

---

## 📌 Step 2: Identify Active Profile per NIC

```powershell
# Which profile is each NIC using?
Get-NetConnectionProfile | Format-Table InterfaceAlias, NetworkCategory

# Common surprise: NIC is on "Public" when customer expects "Domain"
# This happens when domain controller is unreachable on that NIC
```

**Frequent Issue**: Customer sets rules for "Domain" profile, but NIC is on "Public" → rules don't apply!

---

## 📌 Step 3: Find the Blocking Rule

### Method A: Use Firewall Log
```powershell
# Enable logging if not already
Set-NetFirewallProfile -All -LogBlocked True -LogFileName "%systemroot%\system32\LogFiles\Firewall\pfirewall.log" -LogMaxSizeKilobytes 32768

# Read the log (look for DROP entries)
Get-Content "$env:SystemRoot\system32\LogFiles\Firewall\pfirewall.log" -Tail 50 | 
    Select-String "DROP"
```

Log format: `date time action protocol src-ip dst-ip src-port dst-port`

### Method B: Use Auditing (Event Log)
```powershell
# Enable firewall auditing
auditpol /set /subcategory:"Filtering Platform Packet Drop" /success:enable /failure:enable
auditpol /set /subcategory:"Filtering Platform Connection" /success:enable /failure:enable

# Check Event Viewer: Security log, Event ID 5157 (blocked connection)
Get-WinEvent -FilterHashtable @{LogName='Security'; Id=5157} -MaxEvents 10 |
    Format-List TimeCreated, Message
```

### Method C: WFP Diagnostic (Advanced)
```powershell
# Capture WFP state — shows EXACTLY which filter blocked traffic
netsh wfp capture start
# Reproduce the issue...
netsh wfp capture stop

# Output: wfpdiag.cab → extract and analyze wfpdiag.xml
```

## 📌 Step 4: Local Policy vs GPO

```powershell
# Check if GPO is managing firewall
gpresult /h C:\temp\gpresult.html
# Open the HTML → look for "Windows Defender Firewall" settings

# Check registry for GPO-pushed rules
reg query "HKLM\SOFTWARE\Policies\Microsoft\WindowsFirewall" /s

# List GPO-configured rules vs local rules
Get-NetFirewallRule | Where-Object { $_.PolicyStoreSource -eq "GroupPolicy" } |
    Format-Table DisplayName, Direction, Action
```

**Key Insight**: GPO rules **merge** with local rules (by default). But if GPO sets "Apply local firewall rules: No" → local rules are IGNORED.

## 📌 Common Scenarios & Fixes

### Scenario 1: App blocked after Windows Update
- Windows Update may reset firewall rules or profiles
- Check: `Get-NetFirewallRule -DisplayName "*AppName*"` — is the rule still there?

### Scenario 2: RDP blocked
```powershell
# Check RDP firewall rule
Get-NetFirewallRule -DisplayGroup "Remote Desktop" | 
    Format-Table DisplayName, Enabled, Profile, Action

# Enable if disabled
Enable-NetFirewallRule -DisplayGroup "Remote Desktop"
```

### Scenario 3: Ping (ICMP) blocked
```powershell
# Check ICMPv4 rules
Get-NetFirewallRule -DisplayName "*ICMPv4*" | 
    Format-Table DisplayName, Enabled, Action, Profile

# Enable ICMP Echo Request
Enable-NetFirewallRule -Name "CoreNet-Diag-ICMP4-EchoRequest-In"
```

### Scenario 4: Firewall service won't start
```powershell
# Check BFE and MPSSVC
Get-Service BFE, MPSSVC | Format-Table Name, Status, StartType

# BFE must start BEFORE MPSSVC
# If BFE won't start → check for corrupt WFP drivers
# sfc /scannow and DISM repair may help
```
