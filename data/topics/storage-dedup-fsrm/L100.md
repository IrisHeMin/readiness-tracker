## 1. Data Deduplication Overview

Data Deduplication (dedup) is a Windows Server role service that identifies and eliminates
duplicate data on a volume **without compromising data integrity**. It works post-process —
files are written normally, then an optimization job finds redundant chunks and replaces
them with references to a single stored copy.

**How it works — the pipeline:**

```
┌────────────┐     ┌────────────────┐     ┌──────────────┐     ┌──────────────────────┐
│  Files on   │ ──▶ │  Variable-size  │ ──▶ │  Hash each    │ ──▶ │  Dedup Store          │
│  Volume     │     │  Chunking       │     │  chunk (SHA)  │     │  (unique chunks only) │
└────────────┘     └────────────────┘     └──────────────┘     └──────────────────────┘
                                                                         │
                                                                         ▼
                                                               ┌──────────────────────┐
                                                               │  Stream Map           │
                                                               │  (file → chunk refs)  │
                                                               └──────────────────────┘
```

- **Chunking** — files are split into variable-size chunks (32 KB–128 KB).
- **Hashing** — each chunk gets a SHA-256-based hash; identical hashes = duplicate data.
- **Dedup Store** — only unique chunks are kept in `System Volume Information\Dedup\`.
- **Stream Map** — each file keeps a map of chunk references instead of raw data.

Typical savings: **50–90 %** for general file server workloads, VDI, and backup volumes.

---

## 2. Supported Workloads

| Workload | Usage Type | Typical Savings | Notes |
|---|---|---|---|
| General Purpose File Server | `Default` | 50–70 % | User shares, department folders, software distribution |
| Hyper-V (VDI) | `HyperV` | 80–95 % | Virtual Desktop Infrastructure with similar VMs |
| Backup / Virtualized Backup | `Backup` | 50–80 % | DPM, VEEAM, Windows Server Backup targets |

**Supported volumes:** NTFS data volumes, ReFS (Server 2019+), Cluster Shared Volumes on S2D.

**Excluded volumes:**
- Boot / System volumes
- CSV volumes that are **not** running Storage Spaces Direct (S2D)
- Volumes smaller than the minimum (no hard minimum but very small volumes show no benefit)
- Read-only or removable media

## 3. Dedup Components and Jobs

| Job | Purpose | Default Schedule |
|---|---|---|
| **Optimization** | Finds and deduplicates duplicate chunks across files | Every hour (background, low priority) |
| **Garbage Collection** | Removes unreferenced chunks from the chunk store | Weekly (Saturday 02:35 AM) |
| **Integrity Scrubbing** | Verifies chunk store integrity and repairs corruption | Weekly (Saturday 03:35 AM) |

- Optimization runs as a **background** job with low I/O priority to avoid impacting production.
- Garbage Collection reclaims space from deleted or modified files.
- Scrubbing uses checksums to detect and (where possible) repair silent data corruption.

```powershell
# View scheduled dedup jobs
Get-DedupSchedule

# Manually run garbage collection now
Start-DedupJob -Volume D: -Type GarbageCollection
```

## 4. FSRM Overview

**File Server Resource Manager (FSRM)** is a suite of tools for managing and classifying
data stored on file servers. Core capabilities:

- **Quotas** — limit or monitor space usage per folder
- **File Screens** — block or monitor specific file types
- **Storage Reports** — generate reports on disk usage, file types, duplicates
- **File Management Tasks** — automate actions on files (expire, move, classify)

```powershell
# Install FSRM role
Install-WindowsFeature FS-Resource-Manager -IncludeManagementTools
```

---

## 5. FSRM Quotas

| Quota Type | Behavior | Use Case |
|---|---|---|
| **Hard Quota** | Enforces limit — denies writes when exceeded | Prevent volume from filling up |
| **Soft Quota** | Monitors only — sends alerts but allows writes | Capacity monitoring and planning |

Key concepts:
- **Template-based** — create a template once, apply to many paths.
- **Path-based** — quotas are assigned to a specific folder path.
- **Auto-apply quotas** — automatically creates quotas for new subfolders as they are created.

```powershell
# Create a hard quota from a template
New-FsrmQuota -Path "D:\Shares\Marketing" -Template "200 MB Limit"

# Create an auto-apply quota (applies to each new subfolder)
New-FsrmAutoQuota -Path "D:\UserFolders" -Template "1 GB Limit"
```

---

## 6. FSRM File Screens

| Screen Type | Behavior | Use Case |
|---|---|---|
| **Active Screening** | Blocks users from saving restricted file types | Prevent .mp3, .exe, .torrent on file shares |
| **Passive Screening** | Allows the save but generates alerts/events | Monitor for policy violations |

- **File Groups** define collections of file name patterns (e.g., `Audio and Video Files`
  includes `*.mp3`, `*.avi`, `*.mkv`).
- Templates simplify management — define once, apply to multiple folders.

```powershell
# Create an active file screen blocking executables
New-FsrmFileScreen -Path "D:\Shares\Documents" -Template "Block Executable Files"

# List existing file screens
Get-FsrmFileScreen
```

## 7. Essential Commands

```powershell
# ── Data Deduplication ──
Enable-DedupVolume -Volume D: -UsageType Default   # Enable dedup
Get-DedupStatus                                      # Current dedup status
Get-DedupVolume                                      # Volume dedup configuration
Start-DedupJob -Volume D: -Type Optimization         # Run optimization now
Get-DedupJob                                         # View running/queued jobs

# ── FSRM ──
Get-FsrmQuota                                        # List all quotas
New-FsrmQuota -Path "E:\Data" -Size 10GB             # Create a hard quota
Get-FsrmFileScreen                                   # List all file screens
Get-FsrmSetting                                      # View FSRM global settings
```
