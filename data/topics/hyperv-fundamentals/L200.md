## 📌 VM Won't Start — Systematic Diagnosis

When a VM fails to start, work through these categories:

### Category 1: Resource Conflicts

```powershell
# Check if VM is stuck in a saved/paused state
Get-VM -Name "ProblemVM" | Select-Object Name, State, Status

# Check available memory on the host
Get-VMHostNumaNode | Select-Object NodeId, MemoryAvailable, MemoryTotal

# Check if another VM locked the VHD
Get-VM | Get-VMHardDiskDrive | Where-Object { $_.Path -like "*ProblemVM*" }
```

| Symptom | Likely Cause | Resolution |
|---------|-------------|------------|
| "Not enough memory" | Host RAM exhausted or Dynamic Memory minimum too high | Lower `MemoryMinimumBytes` or stop other VMs |
| "The process cannot access the file" | VHD locked by another VM, backup, or AV scan | Check `Get-VMHardDiskDrive` across all VMs; exclude VHD paths from AV |
| "Cannot start because processor is incompatible" | VM migrated from different CPU generation | Enable Processor Compatibility: `Set-VMProcessor -CompatibilityForMigrationEnabled $true` |
| VM stuck in "Starting" state | `vmwp.exe` crash or hung | Kill orphaned `vmwp.exe`, restart `vmms` service |

### Category 2: Corrupt Configuration

```
VM Configuration Files:
C:\ProgramData\Microsoft\Windows\Hyper-V\
├── Virtual Machines\
│   └── {GUID}.vmcx          ← VM config (binary, Server 2016+)
│   └── {GUID}.xml           ← VM config (XML, Server 2012 R2)
├── Snapshots\
│   └── {GUID}.vmrs          ← Saved state / checkpoint runtime
```

```powershell
# If config is corrupt, try re-registering the VM
# 1. Remove the VM entry (keeps files)
Remove-VM -Name "ProblemVM" -Force

# 2. Re-import from the vmcx file
Import-VM -Path "C:\ProgramData\Microsoft\Windows\Hyper-V\Virtual Machines\{GUID}.vmcx"

# If saved state is corrupt, delete it and cold-start
Remove-VMSavedState -VMName "ProblemVM"
Start-VM -Name "ProblemVM"
```

### Category 3: VHD/VHDX Issues

```powershell
# Check VHD health
Test-VHD -Path "C:\VMs\ProblemVM.vhdx"

# Get VHD info (check if it's a differencing disk with a missing parent)
Get-VHD -Path "C:\VMs\ProblemVM.vhdx" | Select-Object VhdType, ParentPath, Size, FileSize

# Compact a bloated dynamic VHDX
Optimize-VHD -Path "C:\VMs\ProblemVM.vhdx" -Mode Full

# Mount a VHD offline for data recovery
Mount-VHD -Path "C:\VMs\ProblemVM.vhdx" -ReadOnly
```

> **Common trap:** A differencing disk whose parent VHD was moved or renamed will fail with "The chain of virtual hard disks is broken." Fix the parent path with `Set-VHD -Path child.vhdx -ParentPath newparent.vhdx`.

## 📌 Virtual NIC Offloading: VMQ, DVMQ, and RSC

These offload features can improve performance but also cause connectivity issues.

| Feature | What It Does | Common Problem |
|---------|-------------|----------------|
| **VMQ** (Virtual Machine Queue) | Maps VM traffic to dedicated hardware queue on NIC | NIC firmware bugs → packet drops, black-hole traffic |
| **DVMQ** (Dynamic VMQ) | Dynamically adjusts VMQ queue count | Imbalanced CPU load, poor throughput |
| **RSC** (Receive Segment Coalescing) | Coalesces small packets into larger ones | Breaks packet inspection tools, incorrect MTU behavior |
| **SR-IOV** | Bypasses vSwitch, gives VM direct NIC queue | Driver compatibility issues, breaks NIC teaming |

### Troubleshooting NIC Offload Issues

```powershell
# Disable VMQ on the physical NIC (common first step for NIC issues)
Set-NetAdapterVmq -Name "Ethernet" -Enabled $false

# Disable RSC on the vSwitch
Get-NetAdapterRsc -Name "vEthernet (ExternalSwitch)" | Disable-NetAdapterRsc

# Check VMQ status per adapter
Get-NetAdapterVmq -Name "Ethernet" | Format-List

# Check VMQ queue assignment for VMs
Get-NetAdapterVMQQueue -Name "Ethernet"

# Disable RSS (Receive Side Scaling) if needed
Set-NetAdapterRss -Name "Ethernet" -Enabled $false

# Nuclear option: disable all offloads for troubleshooting
Set-NetAdapterAdvancedProperty -Name "Ethernet" -DisplayName "Large Send Offload V2 (IPv4)" -DisplayValue "Disabled"
Set-NetAdapterAdvancedProperty -Name "Ethernet" -DisplayName "Large Send Offload V2 (IPv6)" -DisplayValue "Disabled"
```

> **Rule of thumb:** If VMs lose network randomly, disable VMQ first. If throughput is poor, check RSC and LSO settings.

## 📌 Virtual Switch Troubleshooting

### Common vSwitch Issues

| Problem | Diagnostic | Fix |
|---------|-----------|-----|
| Host loses network after creating External switch | External switch binds to NIC, removing host IP | Ensure `AllowManagementOS` is `$true`; check vNIC IP config |
| VMs can't reach each other on same switch | VLAN mismatch or MAC address conflict | Check `Get-VMNetworkAdapterVlan`; verify no duplicate MACs |
| VM has no connectivity after NIC teaming | Team bound to wrong switch | Recreate switch on the NIC team; check `Get-NetLbfoTeam` |
| Performance drops on switch | Switch Extension (monitoring/filter) causing overhead | Check `Get-VMSwitchExtension`; disable suspect extensions |

```powershell
# Get detailed switch info
Get-VMSwitch | Format-List *

# Check switch extensions (antivirus, monitoring agents can cause issues)
Get-VMSwitchExtension -VMSwitchName "ExternalSwitch"

# Check VLAN configuration on VM NICs
Get-VMNetworkAdapterVlan -VMName "DC01"

# Set VLAN for a VM
Set-VMNetworkAdapterVlan -VMName "DC01" -Access -VlanId 100

# Reset a VM network adapter (clears MAC, VLAN, IP settings)
Remove-VMNetworkAdapter -VMName "ProblemVM" -Name "Network Adapter"
Add-VMNetworkAdapter -VMName "ProblemVM" -SwitchName "ExternalSwitch"
```

## 📌 Hyper-V Replica Troubleshooting

Hyper-V Replica provides asynchronous VM replication for disaster recovery.

```
Primary Server ──── Replication (HTTP/HTTPS) ────► Replica Server
  (Source VM)          Every 30s / 5min / 15min       (Replica VM)
```

### Common Replica Issues

| Problem | Check | Resolution |
|---------|-------|------------|
| Replication failed: "Access denied" | Firewall / auth config | Open TCP 80 (HTTP) or 443 (HTTPS); check Kerberos/cert auth |
| Replication suspended | `Get-VMReplication` shows "Error" | Check `Get-VMReplicationServer`; Resume with `Resume-VMReplication` |
| High replication latency | WAN bandwidth / VHD change rate | Increase replication interval; compress; exclude temp VHDs |
| Test failover fails | Replica VM has IP conflicts | Use isolated test network for test failover |

```powershell
# Check replication health
Get-VMReplication | Format-Table VMName, State, Health, Mode, FrequencySec

# View replication statistics
Measure-VMReplication -VMName "DC01"

# Resume suspended replication
Resume-VMReplication -VMName "DC01"

# Force resync after extended outage
Start-VMInitialReplication -VMName "DC01"
```

## 📌 Memory Management: Dynamic Memory & Smart Paging

### Dynamic Memory

```
┌─────────────────────────────────────────┐
│  MemoryMaximum (ceiling)      8 GB      │
│  ┌───────────────────────────────────┐  │
│  │  Currently Assigned        5 GB   │  │
│  │  ┌───────────────────────────┐    │  │
│  │  │  MemoryDemand      4 GB  │    │  │
│  │  └───────────────────────────┘    │  │
│  └───────────────────────────────────┘  │
│  MemoryMinimum (floor)        2 GB      │
│  MemoryBuffer (% headroom)    20%       │
└─────────────────────────────────────────┘
```

```powershell
# Configure Dynamic Memory
Set-VM -Name "DC01" -DynamicMemory `
    -MemoryMinimumBytes 2GB `
    -MemoryStartupBytes 4GB `
    -MemoryMaximumBytes 8GB

# Check current memory pressure
Get-VM | Select-Object Name, MemoryAssigned, MemoryDemand, MemoryStatus

# Memory status values: OK, Low, Warning
```

### Smart Paging

Smart Paging activates **only during VM restart** when the host cannot satisfy the `MemoryStartupBytes` but could satisfy `MemoryMinimumBytes`. It pages to disk temporarily.

```powershell
# Set smart paging file location (use fast storage!)
Set-VM -Name "DC01" -SmartPagingFilePath "D:\SmartPaging"
```

> **Trap:** Smart Paging is NOT a substitute for adequate RAM. It only covers the gap during restart. If it activates frequently, add more host memory.

## 📌 Checkpoints: Production vs Standard

| Feature | Standard Checkpoint | Production Checkpoint |
|---------|-------------------|-----------------------|
| **Method** | Saves entire VM state (RAM + disk) | Uses VSS (Windows) or fsfreeze (Linux) |
| **Application consistent** | ❌ No (point-in-time memory snapshot) | ✅ Yes (like a proper backup) |
| **Restoring to checkpoint** | VM resumes exact state (apps mid-operation) | VM boots fresh with consistent disk |
| **Use case** | Dev/test, debugging exact state | Production backup before patching |
| **Default in 2016+** | No | ✅ Yes (default) |

```powershell
# Set checkpoint type
Set-VM -Name "DC01" -CheckpointType Production      # Recommended
Set-VM -Name "DC01" -CheckpointType Standard         # Dev/test only
Set-VM -Name "DC01" -CheckpointType ProductionOnly   # Fail if VSS unavailable

# Create a checkpoint
Checkpoint-VM -Name "DC01" -SnapshotName "Pre-Patch $(Get-Date -Format 'yyyyMMdd')"

# List checkpoints
Get-VMCheckpoint -VMName "DC01"

# Revert to checkpoint
Restore-VMCheckpoint -VMName "DC01" -Name "Pre-Patch 20240101"

# Delete all checkpoints (merges differencing disks)
Remove-VMCheckpoint -VMName "DC01" -Name *
```

> **Warning:** Checkpoint disks are differencing VHDs. Too many checkpoints → disk I/O penalty and fragmentation. Always merge (delete) old checkpoints.

## 📌 Key Event Logs & Diagnostic Tools

### Event Log Locations

| Log Path | What It Contains |
|----------|-----------------|
| `Microsoft-Windows-Hyper-V-VMMS-Admin` | VM management events (start, stop, create, delete) |
| `Microsoft-Windows-Hyper-V-Worker-Admin` | Per-VM worker process events (crashes, device errors) |
| `Microsoft-Windows-Hyper-V-VMSP-Admin` | VM security process events (Secure Boot, shielded VMs) |
| `Microsoft-Windows-Hyper-V-Hypervisor-Admin` | Hypervisor-level events (scheduler, memory) |
| `Microsoft-Windows-Hyper-V-Config-Admin` | VM configuration changes |
| `Microsoft-Windows-Hyper-V-VmSwitch-Operational` | Virtual switch events |
| `Microsoft-Windows-Hyper-V-StorageVSP-Admin` | Storage virtualization events |

```powershell
# Get recent Hyper-V errors
Get-WinEvent -LogName "Microsoft-Windows-Hyper-V-VMMS-Admin" -MaxEvents 20 |
    Where-Object { $_.LevelDisplayName -eq "Error" } |
    Format-Table TimeCreated, Id, Message -Wrap

# Get VM worker errors for a specific VM
Get-WinEvent -LogName "Microsoft-Windows-Hyper-V-Worker-Admin" -MaxEvents 50 |
    Where-Object { $_.Message -like "*ProblemVM*" }

# Export all Hyper-V logs for support case
Get-WinEvent -ListLog "Microsoft-Windows-Hyper-V*" |
    ForEach-Object { wevtutil epl $_.LogName "$($_.LogName -replace '/','-').evtx" }
```

### Diagnostic Quick Reference

```powershell
# Overall host health check
Get-VMHost | Select-Object ComputerName, LogicalProcessorCount,
    MemoryCapacity, VirtualHardDiskPath, VirtualMachinePath

# Check Hyper-V services
Get-Service vmms, vmcompute | Format-Table Name, Status, StartType

# Performance counters
Get-Counter "\Hyper-V Hypervisor Logical Processor(_Total)\% Total Run Time"
Get-Counter "\Hyper-V Dynamic Memory Balancer(*)\Available Memory"
```
