## 📌 What is Hyper-V?

Hyper-V is Microsoft's **Type 1 (bare-metal) hypervisor** — it runs directly on hardware, not on top of a host OS. When you enable Hyper-V on Windows, the OS itself becomes a **parent partition** running on the hypervisor.

```
┌─────────────────────────────────────────────────────┐
│                    HARDWARE (CPU, RAM, NIC, Disk)    │
├─────────────────────────────────────────────────────┤
│              HYPERVISOR (hvix64.exe / hvax64.exe)    │
├──────────────────┬──────────────┬───────────────────┤
│  Parent Partition │ Child Part 1 │  Child Part 2     │
│  (Root / Host)    │   (VM #1)    │    (VM #2)        │
│                   │              │                   │
│  Windows Server   │  Win 2022    │   Ubuntu 22.04    │
│  VMM Service      │  IC Drivers  │   IC Drivers      │
│  WMI Provider     │  Apps        │   Apps            │
│  VS Provider      │              │                   │
└──────────────────┴──────────────┴───────────────────┘
```

> **Key insight:** Even after enabling Hyper-V, the "host OS" is actually running inside the parent partition. The hypervisor sits below everything.

## 📌 Hyper-V Core Components

| Component | Process / Service | Role |
|-----------|------------------|------|
| **Hypervisor** | `hvix64.exe` (Intel) / `hvax64.exe` (AMD) | CPU scheduling, memory partitioning, intercept handling |
| **VM Management Service** | `vmms.exe` | VM lifecycle (create, delete, configure), WMI provider |
| **VM Worker Process** | `vmwp.exe` (one per VM) | Emulates devices, handles I/O for a specific VM |
| **VM Compute** | `vmcompute.exe` | Compute fabric used by containers and VMs (Server 2016+) |
| **Virtual Machine Bus** | `vmbus.sys` | High-speed communication channel between parent & child partitions |
| **Integration Services** | `vmicXXX.sys` drivers | Guest-to-host cooperation services |

### Parent Partition vs Child Partitions

```
┌─────────────────────────────────────┐
│         Parent Partition             │
│  ┌──────────┐  ┌──────────────────┐ │
│  │ vmms.exe │  │  Device Drivers  │ │
│  │ (VM Mgmt)│  │  (Real HW access)│ │
│  └──────────┘  └──────────────────┘ │
│         ▲  VMBus  ▲                  │
├─────────┼─────────┼──────────────────┤
│         ▼         ▼                  │
│  ┌────────────────────────┐          │
│  │    Child Partition      │          │
│  │  vmwp.exe (worker)     │          │
│  │  Synthetic drivers     │          │
│  │  Integration Services  │          │
│  └────────────────────────┘          │
└─────────────────────────────────────┘
```

- **Parent partition** — has direct hardware access, runs `vmms.exe`, hosts the Virtualization Service Providers (VSPs)
- **Child partitions** — VMs that use Virtualization Service Clients (VSCs) to communicate via VMBus

## 📌 VM Generations: Gen 1 vs Gen 2

| Feature | Gen 1 | Gen 2 |
|---------|-------|-------|
| **Firmware** | BIOS (legacy) | UEFI |
| **Secure Boot** | ❌ Not supported | ✅ Supported |
| **Boot from SCSI** | ❌ IDE only | ✅ SCSI boot |
| **Boot from VHD format** | VHD or VHDX | VHDX only |
| **PXE boot** | Legacy network adapter | Synthetic network adapter |
| **Max RAM** | 1 TB | 12 TB |
| **IDE controller** | ✅ Emulated | ❌ Removed |
| **COM / Serial ports** | ✅ Emulated | ❌ Removed |
| **Floppy controller** | ✅ Emulated | ❌ Removed |
| **Linux guest support** | ✅ Yes | ✅ Yes (most modern distros) |
| **Resize boot VHDX online** | ❌ No | ✅ Yes |
| **Hot add/remove SCSI** | ✅ Yes | ✅ Yes |

> **Rule of thumb:** Use **Gen 2** for all new VMs unless you specifically need legacy BIOS or unsupported guest OS.

## 📌 Integration Services (IC)

Integration Services enable cooperative communication between host and guest. They require IC drivers inside the guest OS.

| Service | Component | Purpose |
|---------|-----------|---------|
| **Heartbeat** | `vmicheartbeat` | Host detects if guest OS is responsive |
| **Time Synchronization** | `vmictimesync` | Syncs guest clock with host (critical for AD/Kerberos) |
| **Key-Value Pair Exchange** | `vmickvpexchange` | Pass data between host and guest via registry/KVP |
| **VSS (Volume Shadow Copy)** | `vmicvss` | Application-consistent backups of the guest |
| **Guest Shutdown** | `vmicshutdown` | Graceful OS shutdown from host |
| **Guest Services** | `vmicguestinterface` | Copy files to the guest without network (Copy-VMFile) |

Check IC status with PowerShell:
```powershell
# List all integration services for a VM
Get-VMIntegrationService -VMName "DC01"

# Enable a specific service
Enable-VMIntegrationService -VMName "DC01" -Name "Guest Service Interface"

# Check IC version inside the guest (run in guest)
(Get-ItemProperty "HKLM:\SOFTWARE\Microsoft\Virtual Machine\Auto").IntegrationServicesVersion
```

## 📌 Virtual Switch Types

Hyper-V virtual switches connect VMs to networks. There are three types:

| Switch Type | VM ↔ VM | VM ↔ Host | VM ↔ External Network | Use Case |
|-------------|---------|-----------|----------------------|----------|
| **External** | ✅ | ✅ | ✅ | Production VMs needing LAN/internet |
| **Internal** | ✅ | ✅ | ❌ | Host-to-VM communication only |
| **Private** | ✅ | ❌ | ❌ | Isolated VM-to-VM lab environments |

```
External Switch          Internal Switch         Private Switch
┌──────────────┐        ┌──────────────┐        ┌──────────────┐
│  Physical NIC │        │   No NIC     │        │   No NIC     │
│  (bound)      │        │              │        │              │
├──────────────┤        ├──────────────┤        ├──────────────┤
│  vSwitch      │        │  vSwitch     │        │  vSwitch     │
├───┬──────┬───┤        ├───┬──────┬───┤        ├───┬──────┬───┤
│VM1│ VM2  │Host│       │VM1│ VM2  │Host│       │VM1│ VM2  │    │
│   │      │vNIC│       │   │      │vNIC│       │   │      │    │
└───┴──────┴───┘        └───┴──────┴───┘        └───┴──────┴───┘
 ↕ LAN/Internet          ↕ Host only              Isolated
```

```powershell
# Create an external virtual switch
New-VMSwitch -Name "ExternalSwitch" -NetAdapterName "Ethernet" -AllowManagementOS $true

# Create an internal switch
New-VMSwitch -Name "InternalSwitch" -SwitchType Internal

# Create a private switch
New-VMSwitch -Name "PrivateSwitch" -SwitchType Private

# List all switches
Get-VMSwitch | Format-Table Name, SwitchType, NetAdapterInterfaceDescription
```

## 📌 Essential Hyper-V PowerShell Commands

```powershell
# List all VMs with status
Get-VM | Format-Table Name, State, CPUUsage, MemoryAssigned, Uptime

# Start / Stop VMs
Start-VM -Name "DC01"
Stop-VM -Name "DC01"                   # Graceful shutdown (needs IC)
Stop-VM -Name "DC01" -TurnOff          # Force power-off (like pulling plug)
Stop-VM -Name "DC01" -Save             # Save state (hibernate)

# Create a new Gen 2 VM
New-VM -Name "TestVM" `
       -Generation 2 `
       -MemoryStartupBytes 4GB `
       -NewVHDPath "C:\VMs\TestVM.vhdx" `
       -NewVHDSizeBytes 60GB `
       -SwitchName "ExternalSwitch"

# Set VM processors and memory
Set-VM -Name "TestVM" -ProcessorCount 4 -DynamicMemory -MemoryMinimumBytes 2GB -MemoryMaximumBytes 8GB

# Connect an ISO to the VM
Add-VMDvdDrive -VMName "TestVM" -Path "C:\ISO\WindowsServer2022.iso"

# Get VM network adapters
Get-VMNetworkAdapter -VMName "DC01"

# Export a VM for backup/migration
Export-VM -Name "DC01" -Path "D:\VMBackups"
```

## ✅ L100 Checkpoint — You Should Now Know:

1. Hyper-V is a **Type 1 hypervisor** that sits below the host OS (parent partition)
2. Key processes: `vmms.exe` (management), `vmwp.exe` (per-VM worker), `vmbus.sys` (communication)
3. **Gen 2 VMs** use UEFI, support Secure Boot, and boot from SCSI — prefer Gen 2 for new deployments
4. **Integration Services** provide heartbeat, time sync, KVP exchange, VSS, and guest shutdown
5. Three virtual switch types: **External** (full network), **Internal** (host+VM), **Private** (VM-only)
6. Core commands: `Get-VM`, `Start-VM`, `Stop-VM`, `New-VM`, `Get-VMSwitch`
