## 1. Windows Storage Stack Overview

Understanding how I/O flows from application to physical disk is essential for diagnosing storage issues.

```
 ┌─────────────────────────┐
 │      Application         │  ← CreateFile / ReadFile / WriteFile
 ├─────────────────────────┤
 │   Filesystem (NTFS/ReFS) │  ← Translates file ops → LBA offsets
 ├─────────────────────────┤
 │     Volume Manager       │  ← volmgr.sys — maps volumes to partitions
 ├─────────────────────────┤
 │    Partition Manager     │  ← partmgr.sys — reads MBR/GPT tables
 ├─────────────────────────┤
 │    Disk Class Driver     │  ← disk.sys — SCSI command generation
 ├─────────────────────────┤
 │   Port Driver (Storport) │  ← storport.sys — queue management, DMA
 ├─────────────────────────┤
 │      Miniport Driver     │  ← Vendor HBA driver (e.g., LSI, QLogic)
 ├─────────────────────────┤
 │      Physical Disk       │  ← HDD / SSD / NVMe / SAN LUN
 └─────────────────────────┘
```

| Layer | Driver | Role |
|---|---|---|
| Filesystem | ntfs.sys / refs.sys | Metadata, journaling, file-to-LBA mapping |
| Volume Manager | volmgr.sys | Aggregates partitions into volumes |
| Partition Manager | partmgr.sys | Reads partition tables (MBR/GPT) |
| Disk Class | disk.sys | Constructs SCSI CDBs for read/write |
| Port Driver | storport.sys | I/O queuing, DMA mapping, interrupt handling |
| Miniport | Vendor-specific | Communicates with HBA hardware |

## 2. Disk Types: Basic vs Dynamic

| Feature | Basic Disk | Dynamic Disk |
|---|---|---|
| Partition types | Primary, Extended, Logical | Simple, Spanned, Striped, Mirrored, RAID-5 |
| Max partitions (MBR) | 4 primary (or 3+1 extended) | Up to 2000 volumes |
| Cluster Shared Volumes | ✅ Supported | ❌ Not supported |
| Failover Clustering | ✅ Supported | ❌ Not supported |
| Portable across machines | ✅ Yes | ⚠️ Requires import |

### MBR vs GPT Comparison

| Feature | MBR | GPT |
|---|---|---|
| Max partitions | 4 primary | 128 (Windows default) |
| Max disk size | 2 TB | 18.8 EB (exabytes) |
| Boot mode | BIOS / Legacy | UEFI required |
| UEFI support | ❌ No | ✅ Yes |
| Redundancy | Single partition table | Primary + backup header |
| Protective MBR | N/A | ✅ Prevents legacy tools from overwriting |

> 💡 **Tip:** Use `Get-Disk | Select Number, PartitionStyle, Size` to quickly identify MBR vs GPT.

## 3. Partition and Volume Management

### Diskpart Commands (Interactive)

```cmd
diskpart
  list disk
  select disk 0
  list partition
  select partition 2
  extend size=10240        REM Extend by 10 GB
  shrink desired=5120      REM Shrink by 5 GB
  create partition primary size=20480
  format fs=ntfs label="Data" quick
```

### PowerShell Equivalents (Scriptable)

```powershell
# Discovery
Get-Disk | Format-Table Number, FriendlyName, PartitionStyle, Size, OperationalStatus
Get-Partition -DiskNumber 0 | Format-Table PartitionNumber, DriveLetter, Size, Type
Get-Volume | Format-Table DriveLetter, FileSystemLabel, FileSystem, SizeRemaining, Size

# Create and format a new partition
New-Partition -DiskNumber 1 -UseMaximumSize -AssignDriveLetter | Format-Volume -FileSystem NTFS -NewFileSystemLabel "Data"

# Resize a partition (extend or shrink)
Resize-Partition -DiskNumber 0 -PartitionNumber 2 -Size (50GB)
```

> 📎 **Disk Management MMC:** `diskmgmt.msc` provides a GUI view — useful for quick visualization but limited in scripting and S2D environments.

## 4. VHD / VHDX Management

### VHD vs VHDX Comparison

| Feature | VHD | VHDX |
|---|---|---|
| Max size | 2 TB | 64 TB |
| Logical sector size | 512 bytes only | 512 bytes or 4 KB |
| Block size | Fixed 2 MB | Up to 256 MB (configurable) |
| Corruption protection | ❌ None | ✅ Log-based metadata updates |
| Trim/Unmap support | ❌ No | ✅ Yes |

### Essential PowerShell Commands

```powershell
# Create a new dynamic VHDX (50 GB)
New-VHD -Path "D:\VMs\Data.vhdx" -SizeBytes 50GB -Dynamic

# Create a differencing disk (snapshot-like)
New-VHD -Path "D:\VMs\Child.vhdx" -ParentPath "D:\VMs\Base.vhdx" -Differencing

# Mount a VHDX to the host
Mount-VHD -Path "D:\VMs\Data.vhdx"

# Optimize (reclaim unused space from dynamic VHDX)
Optimize-VHD -Path "D:\VMs\Data.vhdx" -Mode Full

# Dismount when done
Dismount-VHD -Path "D:\VMs\Data.vhdx"
```

> ⚠️ **Production note:** Always use VHDX for Hyper-V workloads. VHD is a legacy format with no resilience features.

## 5. Storage Advisory Concepts

| Area | What to Check | Tool / Command |
|---|---|---|
| Capacity planning | Free space trends, growth rate | `Get-Volume`, monitoring dashboards |
| Disk health | SMART attributes, media errors | `Get-PhysicalDisk \| Get-StorageReliabilityCounter` |
| Fragmentation | Fragmentation % on HDD volumes | `Optimize-Volume -DriveLetter C -Analyze` |
| Firmware | Current firmware version vs vendor-recommended | `Get-PhysicalDisk \| Select FirmwareVersion` |
| Driver currency | Storport miniport driver version | `driverquery /v \| findstr /i stor` |

> 🔑 **Proactive approach:** Schedule weekly `Get-PhysicalDisk` health checks and alert on `OperationalStatus -ne 'OK'`.
